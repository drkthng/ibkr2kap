# Plan 12.1: Streamlit UI App Entrypoint & Data Import

## Objective
Create the main Streamlit application entrypoint and implement the Data Import tab/functionality, connecting the UI to the `pipeline.run_import` service.

## Context
- .gsd/SPEC.md
- .gsd/ARCHITECTURE.md
- src/ibkr_tax/services/pipeline.py

## Tasks

<task type="auto">
  <name>Install Streamlit Dependency</name>
  <files>pyproject.toml, uv.lock</files>
  <action>
    Add `streamlit` to the project dependencies using `uv add streamlit`.
  </action>
  <verify>uv run python -c "import streamlit"</verify>
  <done>Streamlit is successfully installed in the virtual environment.</done>
</task>

<task type="auto">
  <name>Create Streamlit Entrypoint and Import Tab</name>
  <files>src/app.py</files>
  <action>
    Create `src/app.py`.
    - Setup basic page config with `st.set_page_config()`.
    - Create a session management block connecting to `ibkr_tax.db.database`.
    - Create Tab 1: "Data Import".
    - Use `st.file_uploader` for XML and CSV files.
    - Save the uploaded file to a temporary file using Python's `tempfile` module.
    - Call `ibkr_tax.services.pipeline.run_import` passing the temp file path, DB session, and 'xml' or 'csv' depending on the upload.
    - Display the returned JSON status/counts in the UI.
  </action>
  <verify>uv run streamlit run src/app.py (and manually verify it starts up without errors)</verify>
  <done>Streamlit app starts correctly and the Data Import layout is visible.</done>
</task>

## Success Criteria
- [ ] `streamlit` is listed in project dependencies.
- [ ] `src/app.py` is created with a working connection to the local SQLite DB.
- [ ] UI allows uploading a Flex Query file and outputs parsed metrics.
