# Plan 6.1: Data Import Repository

## Objective
Establish the database repository layer to persist validated Pydantic schemas (Account, Trade, CashTransaction) into SQLite while ensuring idempotency and preventing duplicates.

## Context
- .gsd/SPEC.md
- src/ibkr_tax/models/database.py
- src/ibkr_tax/schemas/ibkr.py

## Tasks

<task type="auto">
  <name>Implement Repository Functions</name>
  <files>src/ibkr_tax/db/repository.py</files>
  <action>
    Create a new module `repository.py` in `src/ibkr_tax/db/`.
    Implement `import_accounts(session, accounts: list[AccountSchema])`.
    Implement `import_trades(session, trades: list[TradeSchema])`.
    Implement `import_cash_transactions(session, cash_txs: list[CashTransactionSchema])`.
    Use SQLite's `INSERT ... ON CONFLICT DO NOTHING` (via SQLAlchemy `sqlite.Insert` and `on_conflict_do_nothing()`) to ensure idempotency.
    For `Account`, use `account_id` for conflict resolution.
    For `Trade`, use `ib_trade_id` for conflict resolution.
    For `CashTransaction`, there is no unique constraint on the model yet. Do a pre-flight query to check for existing records (matching account_id, date_time, amount, type, action_id) OR add a unique constraint to the DB model if appropriate, to safely prevent duplicate insertion.
  </action>
  <verify>pytest -k "test_repository" -v</verify>
  <done>Functions can insert records and gracefully ignore duplicates on subsequent runs.</done>
</task>
