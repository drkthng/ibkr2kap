# Plan 7.1: FIFO Engine Core Logic

## Objective
Implement the core FIFO matching engine to process BUY and SELL trades into `FIFOLot` and `Gain` records according to German tax law.

## Context
- .gsd/SPEC.md
- .gsd/ROADMAP.md
- src/ibkr_tax/models/database.py
- .gsd/phases/7/RESEARCH.md

## Tasks

<task type="auto">
  <name>Implement FIFO Matching Service</name>
  <files>src/ibkr_tax/services/fifo.py</files>
  <action>
    Create a `FIFOEngine` class/functions that:
    - Takes a list of `Trade` records.
    - If `buy_sell` == 'BUY': creates a new `FIFOLot` (cost basis converted to EUR).
    - If `buy_sell` == 'SELL': fetches existing open `FIFOLots` for the symbol, ordered by `settle_date` ASC.
    - Iterates through the open lots to satisfy the SELL quantity.
    - Creates `Gain` records for each match, calculating EUR PnL and updating `FIFOLot.remaining_quantity`.
    - Handles fractional quantities precisely using `decimal.Decimal`.
  </action>
  <verify>pytest -k "test_fifo_matching"</verify>
  <done>FIFO engine correctly creates lots and calculates exact PnL for basic buy/sell sequences.</done>
</task>

<task type="auto">
  <name>Create FIFO Engine Unit Tests</name>
  <files>tests/test_fifo.py</files>
  <action>
    Write unit tests for the FIFO logic using an in-memory SQLite database.
    - Test basic matching (1 buy, 1 sell).
    - Test partial fill (1 large buy, multiple small sells).
    - Test spanning lots (multiple small buys, 1 large sell).
    - Verify strict `settle_date` ordering.
    - Verify EUR PnL math using exact `Decimal` types.
  </action>
  <verify>pytest tests/test_fifo.py -v</verify>
  <done>All edge cases of quantity matching and EUR conversion are thoroughly tested and pass.</done>
</task>

## Success Criteria
- [ ] `fifo.py` correctly handles BUYs (creating lots) and SELLs (relieving lots and creating gains).
- [ ] Math is perfectly precise using `decimal.Decimal`.
- [ ] Test coverage for the FIFO logic is comprehensive.
