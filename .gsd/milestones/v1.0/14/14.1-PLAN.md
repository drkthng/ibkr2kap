# Plan 14.1: Database Models for Foreign Currency

<objective>
Introduce database models to track the FIFO cost basis of foreign currencies themselves (like USD pool) to handle § 23 EStG requirements.

Purpose: We need to track the acquisition cost in EUR of foreign currencies to calculate realized gains on currency fluctuations when the currency is disposed of.
Output: Two new ORM models in database.py.
</objective>

<task type="auto">
  <name>Add FX models</name>
  <files>src/ibkr_tax/models/database.py</files>
  <action>
    Add `FXFIFOLot` and `FXGain` models.
    `FXFIFOLot`: tracks pool of foreign currency. Needs `currency` (e.g. USD), `date`, `original_amount`, `remaining_amount`, `cost_basis_total` (in EUR), `cost_basis_per_unit` (in EUR). Needs foreign keys to the trade or cash transaction that acquired it if applicable (optional relationships, but nullable IDs are good for tracing).
    `FXGain`: tracks realized FX PnL. Needs `fx_lot_id`, `date`, `amount_matched`, `disposal_proceeds` (in EUR), `cost_basis_matched` (in EUR), `realized_pnl` (in EUR), `days_held`, `is_taxable_section_23` (boolean: true if days_held <= 365).
    AVOID: Modifying existing Trade/FIFOLot/Gain models. This is a parallel engine.
  </action>
