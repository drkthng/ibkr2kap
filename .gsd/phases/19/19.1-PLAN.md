# Plan 19.1: Missing Cost-Basis Reporting

## Objective
Detect sell-trades that lack corresponding buy-trades (resulting in short positions when no short was intended) and report these missing data points to the user before generating the final Anlage KAP tax report. This helps users identify and resolve missing historical data (e.g., transfers from other brokers) to ensure accurate tax calculations.

## Context

- src/ibkr_tax/services/tax_aggregator.py
- src/app.py
- src/ibkr_tax/schemas/report.py
- src/ibkr_tax/models/database.py

    - **Tests**: Add a test in `test_tax_aggregator.py` that mocks a `FIFOLot` with negative `remaining_quantity` and verifies the warning is properly generated in the report.
