# Plan 37.1: Streamlit UX (Logging & Language)

## Objective
Implement a persistent/visible logging mechanism for background tasks and add a language toggle (EN/DE) to the Streamlit UI, addressing open UX todos.

## Context
- .gsd/SPEC.md
- app.py

## Tasks

<task type="auto">
  <name>Implement Persistent Logging</name>
  <files>app.py</files>
  <action>
    - Replace or supplement existing success/info messages during file import and report generation with `st.status()` or an empty container at the top of the app (`st.empty()`) so messages are always visible.
    - Create a dedicated UI element (e.g. `st.toast` or top-level container) for background operations so the user doesn't have to scroll to see them.
  </action>
  <verify>pytest -v</verify>
  <done>Long-running operations display their status clearly without requiring scrolling to find the success/error message.</done>
</task>

<task type="auto">
  <name>Implement Language Toggle</name>
  <files>app.py</files>
  <action>
    - Add a toggle or selectbox in the Streamlit UI for selecting the language (English / German).
    - Store the selected language in `st.session_state`.
    - Update the main UI strings (tabs, buttons, headers) to use the selected language dynamically. (A simple dictionary or mapping function is sufficient).
  </action>
  <verify>pytest -v</verify>
  <done>User can switch the UI language between English and German and see UI text update accordingly.</done>
</task>

## Success Criteria
- [ ] Status messages for parsing and generation are prominently visible.
- [ ] English/German language toggle works and updates core UI elements.
