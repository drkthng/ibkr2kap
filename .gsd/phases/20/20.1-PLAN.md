# Plan 20.1: Native App Wrapper (pywebview)

<objective>
Implement a lightweight local webview wrapper using Python's `pywebview` so the Streamlit application runs like a native desktop app, preventing it from opening in a regular browser tab and ensuring the background server cleanly shuts down when the user closes the app.
</objective>

<context>
- .gsd/SPEC.md
- src/ibkr_tax/main.py
</context>

<tasks>

<task type="auto">
  <name>Add pywebview dependency</name>
  <files>pyproject.toml</files>
  <action>
    Add `pywebview` to project dependencies using `uv add pywebview`. Ensure it resolves neatly.
  </action>
  <verify>uv pip show pywebview confirms installation.</verify>
  <done>pywebview is present in pyproject.toml and uv.lock.</done>
</task>

<task type="auto">
  <name>Create Launcher Script</name>
  <files>src/ibkr_tax/launcher.py</files>
  <action>
    Create `launcher.py`. This script should:
    1. Import `webview`, `subprocess`, `sys`, `time`, `socket`.
    2. Function `find_free_port()` to dynamically find an available port.
    3. Use `subprocess.Popen` to launch `streamlit run src/ibkr_tax/main.py --server.port {free_port} --server.headless=true`.
    4. Wait briefly for Streamlit to start.
    5. Call `webview.create_window('IBKR2KAP', f'http://localhost:{free_port}', width=1280, height=800)`.
    6. Call `webview.start()`.
    7. After `.start()` blocks and finishes (user closed window), `terminate()` the Streamlit subprocess to ensure clean shutdown.
  </action>
  <verify>Running `uv run python src/ibkr_tax/launcher.py` opens a native window containing the app, and terminating it leaves no dangling streamlit processes.</verify>
  <done>Launcher properly manages Streamlit lifecycle and native window UI.</done>
</task>

</tasks>

- [ ] No dangling subprocesses left behind on close
</success_criteria>
