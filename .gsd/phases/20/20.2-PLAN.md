# Plan 20.2: macOS Smart Launcher Scripts

<objective>
Provide a seamless launch experience for macOS and technical users who don't want a compiled executable but still want "one-click" startup. We'll create a `.command` file that automatically activates the `.venv` and runs the new `launcher.py` (which manages Streamlit).
</objective>

<context>
- .gsd/SPEC.md
- src/ibkr_tax/launcher.py
</context>

<tasks>

<task type="auto">
  <name>Create macOS App Generator</name>
  <files>scripts/create_mac_app.command, scripts/ibkr2kap_runner.sh</files>
  <action>
    Create `scripts/create_mac_app.command` which is a bash script.
    - It should check if `.venv` exists. If not, inform the user to run installation first.
    - If `.venv` exists, it creates an AppleScript application out of `scripts/ibkr2kap_runner.sh` OR simply places a shortcut on the desktop. 
    Actually, a simpler approach: `scripts/ibkr2kap_runner.sh` just does:
      ```bash
      #!/bin/bash
      cd "$(dirname "$0")/.."
      source .venv/bin/activate
      python src/ibkr_tax/launcher.py
      ```
    Make sure to `chmod +x` the script.
    Then `create_mac_app.command` just creates an AppleScript app on the Desktop:
      ```bash
      osacompile -e 'do shell script "cd \\"'"$PWD"'\\" && ./scripts/ibkr2kap_runner.sh > /dev/null 2>&1 &"' -o ~/Desktop/IBKR2KAP.app
      ```
    This gives the user a real `.app` on their Mac desktop that launches our Python environment in the background without keeping a terminal window open!
  </action>
  <verify>Scripts exist and have execution permissions locally.</verify>
  <done>App generation script and runner script are present and correct.</done>
</task>

</tasks>
