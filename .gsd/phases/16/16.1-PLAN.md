# Plan 16.1: Streamlit UI Config & DB Browser

## Objective
Remove the Streamlit top-right menu and "Deploy" button to clean up the UI for the end user. Add a new tab to browse and inspect raw SQLite database tables for data verification.

## Context
- .gsd/SPEC.md
- src/app.py

## Tasks

<task type="auto">
  <name>Configure Streamlit UI</name>
  <files>src/app.py, .streamlit/config.toml</files>
  <action>
    Create `.streamlit/config.toml` with `[client] toolbarMode = "minimal"` to hide the explicit toolbar/deploy button natively.
    Additionally, in `src/app.py`, use `st.markdown(..., unsafe_allow_html=True)` to inject CSS targeting `#MainMenu` and `.stDeployButton` setting `visibility: hidden` for extra robustness across streamlit versions.
  </action>
  <verify>cat .streamlit/config.toml && grep "st.markdown" src/app.py</verify>
  <done>Streamlit Deploy button and Main Menu are hidden.</done>
</task>

<task type="auto">
  <name>Add DB Browser Tab</name>
  <files>src/app.py</files>
  <action>
    Add a 4th tab named "🗄️ Database Browser" to `st.tabs` in `src/app.py`.
    Inside the tab, import pandas.
    Fetch a list of all tables from the `ibkr_tax.db` using `pd.read_sql("SELECT name FROM sqlite_master WHERE type='table';", con=engine)`.
    Filter out sqlite internal tables if necessary.
    Use `st.selectbox` to allow the user to select a table from the list.
    Fetch and display the selected table's contents using `st.dataframe(pd.read_sql(f"SELECT * FROM {selected_table}", con=engine))`.
    Ensure no SQL injection is possible by strictly matching against the fetched table names.
    Add a refresh button to sync the current state.
  </action>
  <verify>pytest && grep "Database Browser" src/app.py</verify>
  <done>User can navigate to the DB Browser tab, select a table and see its raw rows in a dataframe.</done>
</task>

## Success Criteria
- [ ] Deploy button and Main Menu are no longer visible.
- [ ] New "🗄️ Database Browser" tab exists.
- [ ] Raw database tables can be listed and inspected within the UI.
