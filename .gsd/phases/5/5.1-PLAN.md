# Plan 5.1: CSV Parser Implementation

## Objective
Implement a fallback parser for IBKR CSV Activity Statements that reads the multi-section CSV format and outputs the same `AccountSchema`, `TradeSchema`, and `CashTransactionSchema` lists as the XML parser.

## Context
- .gsd/SPEC.md
- .gsd/ROADMAP.md
- src/ibkr_tax/schemas/ibkr.py
- src/ibkr_tax/services/flex_parser.py (for output structure reference)

## Tasks

<task type="auto">
  <name>Implement CSV Parser Service</name>
  <files>src/ibkr_tax/services/csv_parser.py</files>
  <action>
    Create a `CSVActivityParser` class mimicking the structure of `FlexXMLParser`:
    - Accept `csv_content` or `csv_path`.
    - Use Python `csv.reader`.
    - Parse "Account Information" section to get Account ID and base currency.
    - Parse "Trades" section where row type is "Data" into `TradeSchema` objects.
    - Parse cash transactions (e.g., "Dividends", "Withholding Tax", "Brokers Interest Paid", etc.) where row type is "Data" into `CashTransactionSchema` objects.
    - Provide a `parse_all()` method returning a dict with `accounts`, `trades`, and `cash_transactions`.
    - Keep track of the `Header` row for each section to reliably map fields by name rather than hardcoded indices.
  </action>
  <verify>pytest -v tests/test_csv_parser.py</verify>
  <done>CSVActivityParser correctly processes a mock CSV and returns validated schema objects.</done>
</task>
